Chronos-Graph

A bitemporal-native graph database purpose-built for RAG and AI-agent memory. Every fact tracks valid time and transaction time, contradictions are invalidated (not deleted), and retrieval returns LLM-ready, cited context.

面向 RAG 与 AI-agent memory 打造的 bitemporal-native 图数据库。每条 fact 同时记录 valid time 与 transaction time,矛盾事实被 invalidate 而非删除,检索直接返回带 citation、可供 LLM 使用的上下文。

v0.0.1 Rust REST · MCP · Embedded Apache-2.0

What this engine does without an LLM: there is no built-in embedding model, so the SIMILAR(...) operator ranks facts with a real BM25 index over their verbalized text. Everything else — bitemporal storage, atomic upsert, point-in-time queries, provenance/citations, communities, entity resolution, multi-tenancy and durability — is implemented and tested.

没有 LLM 时它能做什么: 引擎不内置 embedding model,因此 SIMILAR(...) 用真实的 BM25 index 对 fact 文本排序。其余能力——bitemporal storage、atomic upsert、point-in-time query、provenance/citation、community、entity resolution、multi-tenancy 与 durability——均已实现并测试。

Quick start快速开始

Build the workspace and run the REST service (defaults to 127.0.0.1:8080).

编译 workspace 并启动 REST service(默认 127.0.0.1:8080)。

cargo run -p chronos-server
# listening on 127.0.0.1:8080
# override: CHRONOS_ADDR=0.0.0.0:9000 cargo run -p chronos-server
# first build compiles librocksdb-sys (slow once, then cached)
CHRONOS_DATA_DIR=./data cargo run -p chronos-server --features rocks

Then write a fact and ask a question:

然后写入一条 fact 并提问:

curl -XPOST localhost:8080/v1/memory -H 'content-type: application/json' \
  -d '{"subject":"Alice","predicate":"lives_in","object":"Beijing","valid_from":1000,"doc":10,"chunk":1}'

curl -XPOST localhost:8080/v1/search -H 'content-type: application/json' \
  -d '{"query":"MATCH (n) WHERE SIMILAR(n,\"Alice\") RETURN CONTEXT(cite = true)"}'
from chronos_graph import ChronosClient

c = ChronosClient("http://127.0.0.1:8080")
c.add_memory("Alice", "lives_in", "Beijing", valid_from=1000, doc=10, chunk=1)
print(c.search('MATCH (n) WHERE SIMILAR(n,"Alice") RETURN CONTEXT(cite = true)')["text"])
import { ChronosClient } from "./client";

const c = new ChronosClient("http://127.0.0.1:8080");
await c.addMemory({ subject:"Alice", predicate:"lives_in", object:"Beijing", validFrom:1000, doc:10, chunk:1 });
const res = await c.search('MATCH (n) WHERE SIMILAR(n,"Alice") RETURN CONTEXT(cite = true)');
console.log(res.text);

Core concepts核心概念

  • Fact — a subject predicate object edge carrying a bitemporal span and a provenance reference (doc, chunk).
  • Fact — 一条 subject predicate object edge,带 bitemporal span 与 provenance reference(doc、chunk)。
  • Valid time — when the fact was true in the real world (you supply valid_from, in epoch milliseconds).
  • Valid time — fact 在现实世界中为真的起点(由你提供 valid_from,单位为 epoch milliseconds)。
  • Transaction time — when the system learned it (assigned automatically on write).
  • Transaction time — 系统获知该 fact 的时刻(写入时自动赋值)。
  • Invalidation — writing a contradicting fact under the unique policy closes the old fact's valid span instead of deleting it, so history is preserved and queryable.
  • Invalidation — 在 unique policy 下写入矛盾 fact 会关闭旧 fact 的 valid span 而非删除,历史得以保留并可查询。
  • Citation — every retrieved line can be traced back to its source doc/chunk.
  • Citation — 每条检索结果都可回溯到来源 doc/chunk。
  • Tenant — an isolation boundary; facts, retrieval, communities and resolution are all scoped to a tenant.
  • Tenant — 隔离边界;fact、retrieval、community 与 resolution 都按 tenant 隔离。

REST API

Base URL defaults to http://127.0.0.1:8080. All bodies are JSON. Add the header X-Tenant-Id: <number> to scope a request to a tenant (omit for the default tenant 0).

Base URL 默认 http://127.0.0.1:8080,请求体均为 JSON。加 header X-Tenant-Id: <number> 可将请求限定到某 tenant(省略则为 default tenant 0)。

POST/v1/search

Run an extended-Cypher query and get cited, point-in-time context.

执行 extended-Cypher query,返回带 citation、point-in-time 正确的 context。

// request
{ "query": "MATCH (n) WHERE SIMILAR(n,\"Alice\") AS OF VALID TIME 1500 RETURN CONTEXT(cite = true)" }

// response
{
  "text": "- Alice lives_in Beijing",
  "citations": [ { "doc": 10, "chunk": 1, "snippet": "Alice lives_in Beijing" } ]
}
GET/v1/communities

Level-0 (connected-component) communities with templated, current-fact summaries — the "global" topic view.

level-0(connected-component)community 及其基于 current fact 的模板化 summary——即「global」主题视图。

// response
{ "communities": [
  { "id": 1, "members": ["Alice","Beijing"], "summary": "Community of 2 entities (Alice, Beijing). Current facts: Alice lives_in Beijing" }
] }
POST/v1/resolve

Entity resolution: detect surface variants and merge them. Set dry_run: true to preview candidates without merging.

Entity resolution:检测同一 entity 的不同写法并合并。设 dry_run: true 仅预览 candidate 而不合并。

// request
{ "threshold": 0.9, "dry_run": false }

// response
{
  "merged": 1,
  "candidates": [ { "keep": 1, "drop": 2, "keep_name": "OpenAI", "drop_name": "OpenAI Inc.", "score": 1.0 } ]
}

Python SDK

Single file, standard-library only (urllib), no dependencies. Copy sdks/python/chronos_graph.py into your project.

单文件、仅依赖 standard library(urllib),无第三方依赖。把 sdks/python/chronos_graph.py 拷进你的项目即可。

from chronos_graph import ChronosClient, ChronosError

client = ChronosClient("http://127.0.0.1:8080")

# Ingest (returns the new edge id) / 写入(返回新的 edge id)
edge = client.add_memory(
    "Alice", "lives_in", "Shanghai",
    valid_from=2000, doc=20, chunk=1,
    policy="unique",      # or "append"
)

# Search -> {"text": ..., "citations": [...]}
result = client.search('MATCH (n) WHERE SIMILAR(n,"Alice lives") RETURN CONTEXT(cite = true)')
print(result["text"])
print(result["citations"])

MCP serverMCP server

A built-in Model Context Protocol server (JSON-RPC 2.0 over stdio) exposes the engine as agent-callable tools, completing multi-hop retrieval server-side.

内置 MCP(Model Context Protocol)server,基于 stdio 上的 JSON-RPC 2.0,把引擎暴露为 agent 可调用的 tool,并在 server 端完成 multi-hop retrieval。

cargo run -p chronos-mcp     # JSON-RPC 2.0 on stdin/stdout

Register it with an MCP-capable client:

在支持 MCP 的 client 中注册:

{
  "mcpServers": {
    "chronos": { "command": "/path/to/target/debug/chronos-mcp" }
  }
}

Embedded (Rust)嵌入式 (Rust)

Run the same kernel in-process for single-node or edge deployments.

在进程内运行同一 kernel,适合单机或 edge 部署。

use chronos_embedded::{Chronos, MemoryRetriever};
use chronos_common::config::EngineConfig;

let chronos = Chronos::open(EngineConfig::default())?;   // durable with the `rocks` feature
chronos.facts.ingest(
    "Alice", "lives_in", "Beijing",
    chronos_common::Timestamp::from_millis(1000),
    chronos_common::DocId::new(10), chronos_common::ChunkId::new(1),
    chronos_temporal::ConflictPolicy::UniqueSubjectPredicate,
)?;

let block = MemoryRetriever::new(&chronos.facts)
    .answer("MATCH (n) WHERE SIMILAR(n,\"Alice\") RETURN CONTEXT(cite = true)")?;
println!("{}", block.text);

Multi-tenancyMulti-tenancy

Every fact carries a TenantId. The tenant filter is pushed into the scan, so a tenant-scoped request can never observe another tenant's facts, and contradiction detection never crosses the boundary.

每条 fact 都带 TenantId。tenant filter 被下推进 scan,因此限定 tenant 的请求绝不会看到其他 tenant 的 fact,contradiction detection 也不会跨 tenant。

curl -XPOST localhost:8080/v1/memory \
  -H 'content-type: application/json' -H 'X-Tenant-Id: 42' \
  -d '{"subject":"Alice","predicate":"lives_in","object":"Tokyo","valid_from":1000,"doc":1,"chunk":1}'

curl -XPOST localhost:8080/v1/search \
  -H 'content-type: application/json' -H 'X-Tenant-Id: 42' \
  -d '{"query":"MATCH (n) WHERE SIMILAR(n,\"Alice\") RETURN CONTEXT(cite = true)"}'

Omitting X-Tenant-Id targets the default tenant (0). The same header works on /v1/communities and /v1/resolve.

省略 X-Tenant-Id 即 default tenant(0)。该 header 同样适用于 /v1/communities 与 /v1/resolve。
